# Gatherlink v0.9.1 Release Notes

Date: 2026-05-20

Gatherlink v0.9.1 is a hardening release for Debian personal/lab users and
small sites. It keeps the v0.9 shape narrow: authenticated multipath UDP
transport, Python-owned policy and helpers, Rust-owned packet execution, and
CLI-first operation.

## What Changed Since v0.9.0

- DNS helper supports direct, Gatherlink-tunnel, and DNS-over-HTTPS upstreams.
- VM acceptance tooling writes both human `report.md` and machine-readable
  `report.json` with stable check states.
- Operator-safe lab bundle tooling can generate the three-node Hyper-V shape,
  preflight configs, and render scoped cleanup commands.
- Signed topology diffing explains generation, node, service, revocation, and
  issuer changes before install.
- Session rotation policy helpers plan receiver-index rotation and fail-closed
  decisions in Python.
- The experimental local status HTTP helper has one guarded write endpoint:
  `POST /v1/services/{name}/close`.
- Release artifact tooling builds a tracked source archive, local Python wheel,
  Debian-oriented Rust helper binary, checksum file, and `docs/user/` Wiki
  payload.

## Tested Gates

The v0.9.1 local release gate is:

- `cargo test --workspace`
- `.venv/bin/pytest -q`
- `.venv/bin/ruff check .`
- `.venv/bin/black --check .`
- `python3 -m compileall -q python tests tools`
- `cargo fmt -- --check`
- `gatherlink lab helpers-smoke`
- `gatherlink lab rust-smoke configs/lab/local-dual-path.json --count 5`
- `gatherlink lab rust-smoke configs/lab/local-dual-path-encrypted.json --count 5`
- `gatherlink lab rust-smoke configs/lab/local-three-path.json --count 5`
- `gatherlink lab shared-sink-smoke configs/lab/local-dual-path-encrypted.json --count 5`
- `tools/release/prepare_artifacts.py --version 0.9.1 --out <output-dir>`
- Python bridge tests proving `quic-datagram` and `http3-datagram` configs fail
  closed before the UDP runner can treat them as UDP.

Prepared VM acceptance tooling remains under `tools/hyperv/` and
`tools/vm_acceptance/`. Real VM runs depend on an operator-supplied inventory
and are not run by default during local release preparation.

## Supported Platform

Debian is the only supported operating-system compatibility target for v0.9.1.
WSL and Hyper-V Debian labs are acceptance environments for Debian behavior, not
separate supported platforms.

## Security Notes

- Authenticated encrypted transport remains the normal secure path.
- Static crypto remains an explicit lab/manual fallback.
- Invalid encrypted and relay packets must be silent on the network and counted
  locally where diagnostics are available.
- No plaintext routing is supported.
- Release artifacts refuse obvious host-local paths and secret-looking files.

## Carrier Notes

UDP remains the implemented baseline carrier. The config model names
`quic-datagram` and `http3-datagram` so operators can validate intent and fail
closed, but real packet-time QUIC DATAGRAM and HTTP/3 DATAGRAM carrier execution
is not yet implemented. Do not treat those carrier modes as production-ready
until the carrier code and labs exist.

## Known Limits

- Direct QUIC DATAGRAM and HTTP/3 DATAGRAM carriers are still implementation
  work, not hidden UDP aliases.
- The local REST helper is experimental, loopback by default, and write access
  expires after the configured window.
- The project has not had an external security audit.
