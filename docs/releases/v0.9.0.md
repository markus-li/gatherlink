# Gatherlink v0.9.0 Release Notes

Date: 2026-05-19

Gatherlink v0.9.0 is the first narrow release for Debian personal/lab users and
small sites.

## What This Release Is

- carrier-aware multipath UDP transport
- compact Rust packet executor with Python-owned policy and lifecycle
- authenticated encrypted path transport as the normal secure mode
- silent-drop behavior for invalid encrypted and relay packets
- no plaintext routing
- relay/routing through authenticated outer routing/relay-hop context
- CLI-first operation
- managed helper slices for SOCKS5 TCP CONNECT, TCP forwarding, DNS direct and
  tunnel upstream, WireGuard planning/transport proof, and the narrow time
  helper

## Release Evidence

The planned v0.9 release gates have passed:

- Rust and Python test suites
- local lab packet-movement checks
- WSL acceptance checks
- Hyper-V two-Debian-VM core acceptance
- DNS tunnel VM proof
- active SOCKS5/TCP stream-helper VM proof
- WireGuard endpoint/UDP transport VM proof
- untrusted three-VM relay/routing WireGuard proof
- one-hour operator soak

The latest living evidence summary is in [`docs/project-living-assessment.md`](../project-living-assessment.md).

## Known Limits

- Debian is the tested platform.
- Real-world testing is still limited. The project has mainly been tested by
  the developer for aggregating a fiber connection and a 5G connection, plus
  local, WSL, VM, soak, and simulated-network checks.
- Gatherlink has not had an external security audit.
- Static crypto is explicit lab/manual fallback only; authenticated sessions are
  the intended secure path.
- REST/status helper work is experimental and local by default.
- Project boundaries are defined in [`docs/architecture/architecture-contract.md`](../architecture/architecture-contract.md).
