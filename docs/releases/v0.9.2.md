# Gatherlink v0.9.2 Release Notes

Date: 2026-05-26

Gatherlink v0.9.2 is a stability, observability, and release-readiness update
for the existing v0.9/v0.9.1 surface. It keeps the product shape narrow:
Debian-first, CLI-first, encrypted multipath UDP transport with Python-owned
policy and Rust-owned packet execution.

## What Changed Since v0.9.1

- Scheduler/runtime telemetry is stricter and more useful:
  - Rust exposes queue depth, queued bytes, oldest queued age, drops, and timing
    facts as compact execution telemetry.
  - Python rejects unreasonable path-latency samples before compiling scheduler
    primitives for Rust.
  - `least_queue` is backed by real queue facts rather than placeholder data.
- The Hyper-V VM acceptance runner can enable live scheduler reapply and verify
  scheduler decision/config-reapply diagnostics.
- QUIC DATAGRAM and HTTP/3 DATAGRAM carrier adapters now expose lifecycle,
  send, receive, connect-failure, and datagram-drop counters.
- Carrier adapters enforce configured max datagram sizes locally and count
  oversized datagram drops.
- Blocking helper CLIs, including SOCKS5, use the shared diagnostics drainer so
  helper JSONL is written while services remain live, not only during shutdown.
- Python runtime rekey coordination can emit stable rekey diagnostics and
  compile a validated replacement authenticated session into runtime AEAD facts.
  Autonomous peer-to-peer live rekey negotiation is still future work.
- `gatherlink config diff` compares redacted canonical or runtime JSON so
  operators can review changes before restart or reapply.
- `gatherlink doctor` checks optional helper/carrier tools, common listener and
  port mistakes, package version agreement, release hygiene, and diagnostics
  JSONL shape without mutating the host.
- Handshake packet compatibility helpers reserve the documented fixed
  `mac1`/`mac2` trailer and cookie-reply shape for future anti-DoS cookies.

The v0.9.2 release-candidate tree passed:

- full Rust workspace tests
- full Python tests
- Ruff, Black, Cargo fmt, and Python compile checks
- local helper smoke

## Supported Platform

Debian is the supported operating-system compatibility target for v0.9.2. WSL
and Hyper-V Debian VMs are acceptance environments for Debian behavior, not
separate supported platforms.

## Security Posture

- Authenticated encrypted transport remains the normal secure path.
- Static crypto remains an explicit lab/manual fallback.
- Invalid encrypted and relay packets should be silent on the network and
  counted locally.
- Relay forwarding uses authenticated outer relay-hop state; relays do not make
  endpoint service/exit decisions.
- Gatherlink has not had an external security audit.

## Known Limits

- Full autonomous peer-to-peer rekey negotiation is not claimed in v0.9.2.
- QUIC DATAGRAM and HTTP/3 DATAGRAM carriers are Python-owned carrier adapters;
  UDP remains the Rust-runner baseline.
- Traefik proxy carrier smoke depends on a local Traefik binary and is not part
  of the mandatory default gate.
- Performance tuning is benchmark-led by design: scheduler and transport
  changes are validated against repeatable lab profiles, WireGuard/userspace
  baselines, and documented acceptance thresholds. Current results and
  comparison rules are maintained in `docs/benchmarks/`.
- Broader dynamic mesh, hosted control plane, GUI, non-Debian platform support,
  and production relay-fabric expansion remain future work.
